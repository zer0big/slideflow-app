## API 키 관리

### ⚠️ 중요 사항
- **절대 소스 코드에 API 키를 하드코딩하지 마세요**
- 모든 민감한 정보는 환경 변수로 관리하세요
- 공개 리포지토리에 API 키가 노출되지 않도록 주의하세요

### 올바른 API 키 설정 방법

#### Azure Static Web Apps에서:
1. Azure Portal 접속
2. Static Web App 리소스로 이동
3. **구성** → **애플리케이션 설정** 선택
4. **+ 추가** 클릭
5. 이름: `GEMINI_API_KEY`, 값: 실제 API 키 입력
6. **저장** 클릭

#### 로컬 개발 환경에서:
로컬에서 테스트할 때는 브라우저 콘솔에서 직접 설정하거나, 
개발용 프록시 서버를 사용하세요.

```javascript
// 브라우저 콘솔에서 임시로 설정 (테스트용)
// 주의: 절대 이 키를 커밋하지 마세요!
```

## API 키가 노출된 경우

만약 실수로 API 키가 GitHub에 노출되었다면:

### 1. 즉시 API 키 폐기
- Google Cloud Console → API 및 서비스 → 사용자 인증 정보
- 노출된 키 삭제 또는 비활성화

### 2. 새 API 키 발급
- 새로운 API 키 생성
- Azure Application Settings에 업데이트

### 3. GitHub 히스토리 정리
```bash
# git-filter-repo 도구 사용 권장
# 또는 리포지토리를 새로 생성
```

### 4. 보안 검토
- 노출 기간 동안의 사용량 확인
- 의심스러운 활동 모니터링
